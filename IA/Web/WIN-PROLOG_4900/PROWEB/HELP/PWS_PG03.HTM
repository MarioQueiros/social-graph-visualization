Introduction

The development and testing of a ProWeb application requires the communication channel inherent in the WWW to be simulated on a stand-alone machine.

The ProWeb toolkit provides such a simulation in two alternate ways depending upon the browser used. For Netscape Navigator the communication channel is seamlessly integrated through DDE whereas for Internet Explorer and all other browsers the communication is simulated by a CUT-AND-PASTE mechanism.





Netscape Navigator

Netscape Navigator exports a DDE interface which allows third-party applications to act as both CLIENTs and SERVERs on the local machine.

To register ProWeb as a SERVER application which handles URL requests do the sequence of operations :-

  1. Start Netscape Navigator.

  2. Start Prolog, load the ProWeb toolkit and load the application code.

  3. Execute the Prolog command proweb_nn_register.

Hereafter, all URL requests in Netscape Navigator will be handled by the ProWeb application code.

At the end of a session the ProWeb handler should be removed by a call to the Prolog command proweb_nn_unregister.





Internet Explorer and Other Browsers

 For Internet Explorer and all other browsers the communication channel is simulated by a CUT-AND-PASTE mechanism. The remainder of this section describes how to use this mechanism for Internet Explorer.

  1. Start Prolog, load the ProWeb toolkit and load the application code.

  2. Start Internet Explorer

  3. Load the start page of the ProWeb application.

  4. Fill in the form and hit the submit button.

  5. Copy the contents of the address bar.

  6. From Prolog, enter the command proweb_browser, paste into the edit field and hit OK.

  7. From Internet Explorer load the temporary file PW_NEXT.HTM.

  8. Repeat steps 4-7

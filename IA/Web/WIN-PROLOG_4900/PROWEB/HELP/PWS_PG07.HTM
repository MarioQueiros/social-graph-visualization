Reference Manual - HTML Prolog Terms

The following list constitutes the HTML objects (represented by Prolog terms) which can be used in the definition of ProWeb forms.
  • ProWeb Question
  • ProWeb Help
  • ProWeb Reply
  • Simple Element
  • Applied Element
  • List
  • Attribute
  • Number
  • Money
  • Date
  • Time
  • Quotation
  • Encoding
  • Writing
  • Verbatim




HTML Object: ProWeb Question

 A ProWeb question can be referenced within the definition of a ProWeb form using the Prolog term :- 
        ? QUESTION
where QUESTION is a Prolog term which unifies with the 1st argument of a user-defined clause for proweb_question/2.

The translation is determined by the 2nd argument of the clause.

Examples

Question Definition Prolog Term HTML Translation 
proweb_question(
    yourname,
    [
     method = input,
     type = atom,
     prefill = 'Your Name',
     rows = 1,
     cols = 12
    ] ).
 ?yourname <INPUT NAME="Q001" TYPE=INPUT ROWS=1 COLS=12 VALUE="Your Name"> 
proweb_question(
    peril(_Property,_Peril),
    [
     method = checkbox,
     prefill = on
    ] ).
 ?peril(3,fire) <INPUT NAME="Q002" TYPE=CHECKBOX CHECKED> 
proweb_question(
    peril(_Property,_Peril),
    [
     method = radio,
     select = ['Yes','No']
    ] ).
 ?peril(3,fire) <INPUT NAME="Q003" TYPE=RADIO VALUE="Yes">Yes <INPUT NAME="Q003" TYPE=RADIO VALUE="No">No 
proweb_question(
    peril(_Property,_Peril),
    [
     method = menubox,
     select = ['Yes','No']
    ] ).
 ?peril(3,fire) <SELECT NAME="Q004"> <OPTION VALUE="Yes">Yes <OPTION VALUE="No">No </SELECT>




HTML Object: ProWeb Help

A help button can be referenced within the definition of a ProWeb form using the Prolog term :- 
        ??? QUESTION
where QUESTION is a Prolog term which unifies with the 1st argument of a user-defined clause for proweb_help/3.

The translation is determined by the 2nd and 3rd arguments of the clause.



HTML Object: ProWeb Reply

A ProWeb reply which has been recorded by a call to proweb_post_reply/2 or proweb_post_unique_reply/2 can be referenced within the definition of a ProWeb form using the Prolog term :- 
        ?? REPLY
where REPLY is a Prolog term which unifies with the 1st argument of the call. The translation is determined by the value of the call's 2nd argument.

Examples

Call In Main Procedure Prolog Term HTML Translation 
proweb_post_reply(
  today,
  [1997,1,11] )
 date(`WW, dth MM, YYYY`)
@
??today		 Saturday, 11th January, 1997 
proweb_post_reply(
  total_premium,
  102030.405 )
 money('$','s,.')
@
??total_premium		 $102,030.41




HTML Object: Simple Element

Certain simple elements such as paragraph (P), line break (BR) and horizontal line (HR) are self-standing and are specified by the Prolog terms :- 
        ELEMENT

        ELEMENT( ATTRIBUTE1, ..., ATTRIBUTEk )
Such terms will be translated into the following HTML text, respectively :- <ELEMENT> <ELEMENT Translation of ATTRUBUTE1 ... Translation of ATTRIBUTEk> In addition, closing elements are specified by the Prolog terms :- 
        / ELEMENT

        / ELEMENT( ATTRIBUTE1, ..., ATTRIBUTEk )
which will be translated into the following HTML text, respectively :- </ELEMENT> </ELEMENT Translation of ATTRUBUTE1 ... Translation of ATTRIBUTEk>

Examples

Prolog Term HTML Translation 
p
<P> 
hr( width='50%' )
<HR WIDTH="50%"> 
/a(name=myanchor)
</A NAME="myanchor">




HTML Object: Applied Element

Any HTML element, with or without attributes, can be applied to another HTML object using the Prolog terms :- 
        ELEMENT @ OBJECT

        ELEMENT( ATTRIBUTE1, ..., ATTRIBUTEk ) @ OBJECT
Such terms will be translated into the following HTML text, respectively :- <ELEMENT> Translation of OBJECT </ELEMENT> <ELEMENT Translation of ATTRUBUTE1 ... Translation of ATTRIBUTEk> Translation of OBJECT </ELEMENT>

Examples

Prolog Term HTML Translation 
b @
'Emboldened'
<B> Emboldened </B> 
b @
i @
'Emboldened and italicised'
<B> <I> Emboldened and italicised </I> </B> 
h2( align=center ) @
`Heading Level 2`
<H2 ALIGN=CENTER> Heading Level 2 </H2>




HTML Object: List

Prolog lists can be displayed in HTML format using the terms :- 
        ELEMENT @ LIST

        ELEMENT( ATTRIBUTE1, ..., ATTRIBUTEk ) @ LIST
where the ELEMENT is any of the recognised HTML list styles :-
dir
Denotes a directory list.
dl
Denotes a definition list. Used for a list of defined terms.
ol
Denotes an ordered list. Each item has a number or letter reference.
ul
Formats lines of text as a bulleted list.
Such terms will be translated into the following HTML code fragments, respectively :- <ELEMENT> <LI>1st item in the LIST <LI>2nd item in the LIST ... <LI>Last item in the LIST </ELEMENT> <ELEMENT Translation of ATTRIBUTE1 ... Translation of ATTRIBUTEk> <LI>1st item in the LIST <LI>2nd item in the LIST ... <LI>Last item in the LIST </ELEMENT>

Examples

Prolog Term HTML Translation 
ul @
[ alpha, beta, gamma ]
<UL> <LI>alpha <LI>beta <LI>gamma </UL> 
dir @
[ alpha, beta, gamma ]
<DIR> <LI>alpha <LI>beta <LI>gamma </DIR> 
ol( start=4, type=i )
@
[ alpha, beta, gamma ]
<OL START=4 TYPE=i> <LI>alpha <LI>beta <LI>gamma </OL> 
dl @
[
  b @ alpha = '1st Greek',
  b @ beta  = '2nd Greek',
  b @ omega = 'Last Greek'
]
<DL> <DT><B>alpha</B> <DD>1st Greek <DT><B>beta</B> <DD>2nd Greek <DT><B>omega</B> <DD>Last Greek </DL>




HTML Object: Attribute

One or many attributes can be attached to HTML elements using the Prolog terms :- 
        ATTRIBUTE = PARAMETER
        
        ATTRIBUTE

Examples

Prolog Term HTML Translation 
onclick = `return alert( 'OK' )`
onClick="return alert( 'OK' )" 
border=1,
frame=box,
rules=rows
BORDER=1 FRAME=BOX RULES=ROWS 
type=checkbox,
checked
TYPE=CHECKBOX CHECKED




HTML Object: Number

A Prolog number can be translated as HTML text with a decimal precision using the terms :- 
        number( DECIMALS ) @ NUMBER

Examples

Prolog Term HTML Translation 
number( 2 ) @ 123.456789
123.46 
number( 9 ) @ 123.456789
123.456789000 
number( 0 ) @ 123.456789
123




HTML Object: Money

A Prolog number can be translated as HTML text in some monetary format and prefixed by some monetary symbol using the term :- 
        money( SYMBOL , FORMAT ) @ NUMBER
The SYMBOL is translated verbatim. The FORMATprescribes whether or not the number is signed, whether or not commas are inserted before every third digit (thousands) and whether or not the pennies are shown. The FORMAT is denoted by one of the following Prolog atoms :-

Format Sign Commas Pennies
's,.' prefix yes yes
's,x' prefix yes no
'sx.' prefix no yes
'sxx' prefix no no
'b,.' bracketts yes yes
'b,x' bracketts yes no
'bx.' bracketts no yes
'bxx' bracketts no no
'x,.' none yes yes
'x,x' none yes no
'xx.' none no yes
'xxx' none no no

Examples

Prolog Term HTML Translation 
money( '$', 's,.' ) @
-10203040506070.8090
-$10,203,040,506,070.81 
money( '$', 'b,x' ) @
-10203040506070.8090
($10,203,040,506,070) 
money( '', 'xxx' ) @
-10203040506070.8090
10203040506070




HTML Object: Date

A date term can be translated as HTML text with a specific format using the term :- 
        date( FORMAT ) @ [ YEAR, MONTH, DAY ]
where YEAR, MONTH and DAY are all integers and the FORMAT is either an atom or a string.

The FORMAT determines the precise HTML text generated. It is parsed for sub-strings which correspond to a format directive.

Directive Parameter Presentation
/yyyy YEAR 4-digit year
/yy YEAR 2-digit year
/MM MONTH January ... December
/M MONTH JAN ... DEC
/mm MONTH 01 ... 12
/m MONTH 1 ... 12
/dth DAY 1st ... 31st
/dd DAY 01 ... 31
/d DAY 1 ... 31
/WW DATE Monday ... Sunday
/W DATE MON ... SUN

Examples

Prolog Term HTML Translation 
date( '/dd/mm/yy' ) @
[ 1997 , 1 , 11 ]
110197 
date( '/dd///mm///yy' ) @
[ 1997 , 1 , 11 ]
11/01/97 
date( '/WW, /dth /MM, /yyyy' ) @
[ 1997 , 1 , 11 ]
Saturday, 11th January, 1997 
date( '/dd///mm///yy falls on a "/WW"' ) @
[ 1997 , 1 , 11 ]
11/01/97 falls on a "Saturday"




HTML Object: Time

A time term can be translated as HTML text with a specific format using the term :- 
        time( FORMAT ) @ [ HOUR, MINUTE, SECOND ]
where HOUR, MINUTE and SECOND are all integers and the FORMAT is either an atom or a string.

The FORMAT determines the precise HTML text generated. It is parsed for sub-strings which correspond to a format directive.

Directive Parameter Presentation
/hh HOUR 00 ... 24
/h HOUR 0 ... 12
/mm MINUTE 00 ... 59
/m MINUTE 0 ... 59
/ss SECOND 00 ... 59
/s SECOND 0 ... 59
/ampm TIME am pm

Examples

Prolog Term HTML Translation 
time( '/hh/mm/ss' ) @
[ 15, 1, 59 ]
150159 
time( '/hh:/mm:/ss' ) @
[ 15, 1, 59 ]
15:01:59 
time( '/h:/m:/s' ) @
[ 15, 1, 59 ]
3:1:59 
time( '/hh00 hours' ) @
[ 15, 1, 59 ]
1500 hours 
time( 'The time /hh:/mm:/ss is "/ampm"' ) @
[ 15, 1, 59 ]
The time 15:01:59 is "pm" 
time( 'The time /hh:/mm:/ss is "/ampm"' ) @
[ 5, 1, 59 ]
The time 05:01:59 is "am" 
time( 'The time /hh:/mm:/ss is "/ampm"' ) @
[ 0, 0, 0 ]
The time 00:00:00 is "pm" 
time( 'The time /hh:/mm:/ss is "/ampm"' ) @
[ 12, 0, 0 ]
The time 12:00:00 is "am" 
time( 'The time /hh:/mm:/ss is "/ampm"' ) @
[ 12, 0, 1 ]
The time 12:00:01 is "pm"




HTML Object: Quotation

An object can be embraced by quotation marks using the term :- 
        quotation( CHAR ) @ OBJECT
which is translated into the following HTML text :- CHAR Translation of OBJECT CHAR where CHAR is the quotation character.

Examples

Prolog Term HTML Translation 
b @
quotation(`"`) @
'Emboldened'
<B> "Emboldened" </B> 
b @
quotation(`"`) @
i @
'Emboldened and italicised ?'
<B> "<I> Emboldened and italicised ? </I>" </B>




HTML Object: Encoding

Prolog atoms, strings and numbers are by default encoded using the ISO-Latin-1 character map which translates certain ASCII characters (e.g. the quotation character ") into symbolic references (e.g. &quot;)

The purpose of this encoding is to stop the client browser from interpreting such characters as HTML and to treat them verbatim.

This default encoding can be over-ridden by using the term :- 

        unencoded @ OBJECT
In addition, other character maps can be specified by using the term :- 
        encoded(HOW) @ OBJECT
where HOW is either a term of the form :- 
        BYTE = OBJECT
which replaces all occurrences of BYTE with the translation of OBJECT, or it is the name of a built-in character map :- 
        javascript
The javascript mapping function translates certain ASCII control characters (0-31) as follows :-

ASCII Byte Code JavaScript Translation
008\b
009\t
013\r

Examples

Prolog Term HTML Translation 
`This is "quoted"`
This is &quot;quoted&quot; 
unencoded @ `This is "quoted"`
This is "quoted" 
encoded(javascript) @
`Line "1"~M~IIndented Line 2`
Line &quot;1&quot;\r\tIndented Line 2 
encoded(13='/R') @
`Line "1"~M~IIndented Line 2`
Line &quot;1&quot;/R~IIndented Line 2




HTML Object: Writing

Prolog atoms, strings and numbers are normally translated using the built-in predicate write/1. This default translation can be over-ridden by using the terms :- 
        write @ OBJECT
        
        writeq @ OBJECT
        
        print @ OBJECT
        
        printq @ OBJECT
        
        display @ OBJECT
        
        write(PREDICATE) @ OBJECT
where PREDICATE/1 is a defined Prolog procedure.

Examples

Prolog Term HTML Translation 
write @ `A` - `B`
A - B 
writeq @ `A` - `B`
`A` - `B` 
display @ `A` - `B`
-(A,B) 
write(print_postfix) @ `A` - `B`
AB-




HTML Object: Verbatim

Any object whatsoever, or indeed any list of objects, can be translated verbatim using the terms :- 
        verbatim @ OBJECT

        verbatims @ LIST
which are translated into the following HTML texts, respectively :- OBJECT LIST[1]LIST[2]...LIST[n] The translations are subject to any encoding in place, such as :- 
        unencoded @ ..... @ verbatim @ OBJECT

        encoded(javascript) @ ..... @ verbatim @ OBJECT
Prior to encoding the translation is performed by calls to the Prolog built-in predicate write/1 unless it occurs within the context of terms such as :- 
        writeq @ ..... @ verbatim @ OBJECT

        display @ ..... @ verbatim @ OBJECT
in which case the translation is performed by calls to the Prolog built-in predicates writeq/1 or display/1.

Examples

Prolog Term HTML Translation 
b @
verbatim @
`This is verbatim`
<B> This is verbatim </B> 
verbatim @
b @
`This is verbatim`
b @ This is verbatim 
verbatims @
[ `These`, `are`, `verbatim` ]
Theseareverbatim 
b @
writeq @
verbatim @
`This is quoted verbatim`
<B> `This is quoted verbatim` </B> 
writeq @
verbatim @
b @
`This is quoted verbatim`
b @ `This is quoted verbatim` 
writeq @
verbatims @
[ `These`, `are`, `quoted verbatim` ]
`These``are``quoted verbatim`
